# controls

## Purpose

Hard controls and non-build boundaries for Tool 2 implementation (C-01–C-10, scope stop rules).

## What belongs here

- Control rule documentation mirrored from DFSS artefacts
- Out-of-scope checks before merge/release
- Regression protection for existing generator (C-09)

## What must not be built here

- Architecture enforcement tooling
- Pricing, CEKS, Bot-Packs, Dashboard

## Status

**Active** (governance documentation)

## Tool 2 connection

Tool 2 must **not** claim:

- automatic employee / project / SDL release
- automatic audit readiness
- full certification readiness

unless required evidence and review status are present. See `docs/03-controls/`.
