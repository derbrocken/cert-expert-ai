# Cert-Expert AI — Placeholder Registry

This is the single authoritative reference for all `{Placeholder}` tokens used across
Tool 1, Tool 2, and all Fachbots. Every template `.docx` file must use only names from
this registry. Every bot output dict must use only names from this registry.

**Convention:** All placeholder tokens are written as `{Key}` in Word templates.
All keys are `PascalCase` except `{currentDate}` (legacy outlier — do not rename).

---

## Layer 1 — Company / Document Metadata (Tool 1)

These placeholders originate from the Standard Model Creator (Tool 1) and apply to
all document types. They are provided at render time by `inputs/company_data.json`
and merged by `shared/docx_builder.py`.

| Token | Key | Type | Required | Format / notes |
|---|---|---|---|---|
| `{Logo}` | `Logo` | Image | No | PNG/JPG/SVG file. Scaled to max 150×60 px. Empty string if absent. |
| `{CompanyName}` | `CompanyName` | String | Yes | Full legal company name |
| `{CompanyStreet}` | `CompanyStreet` | String | No | Street and house number |
| `{CompanyZip}` | `CompanyZip` | String | No | Postal code |
| `{CompanyCity}` | `CompanyCity` | String | No | City |
| `{CompanyCountry}` | `CompanyCountry` | String | No | Country |
| `{CompanyAddressLine}` | `CompanyAddressLine` | String | No | Optional second address line |
| `{CompanyEmail}` | `CompanyEmail` | String | No | Contact email |
| `{DocVersion}` | `DocVersion` | String | Yes | e.g. `"1.0"` |
| `{DocDate}` | `DocDate` | String | Yes | Format: `DD.MM.YYYY` |
| `{CreatedBy}` | `CreatedBy` | String | Yes | Author name |
| `{ApprovedBy}` | `ApprovedBy` | String | No | Approver name — empty string if not yet approved |
| `{currentDate}` | `currentDate` | String | Auto | Auto-injected at render time. Format: `"May 18, 2026"` (long English). **Lowercase 'c' — do not change.** |

**Source:** `app/actions/send-model-entries.ts`, `lib/validations/model-form.ts`

## Rendering Rules

1. **All Layer 1 placeholders** must be provided at render time, even if their value is
   an empty string. Missing keys are left unreplaced in the output DOCX — they appear
   as literal `{Key}` text, which indicates a rendering configuration error.

2. **Empty string is valid** — `{ApprovedBy}: ""` renders as a blank field, which is
   correct for documents not yet approved.

3. **`{currentDate}` is auto-injected** by `shared/docx_builder.py` at render time.
   It is never set by a bot or by `company_data.json`.

4. **`{Logo}` is handled separately** from string placeholders. It is passed as a
   file path in the render context, not as a string in the `placeholders` dict.
   The renderer (`render.js`) converts it to the `easy-template-x` image descriptor.

5. **Namespace isolation** — Bot placeholders (`GB_*`, `SK_*`, etc.) must not appear
   in company_data.json and company placeholders must not appear in bot output dicts.
   `shared/docx_builder.py` merges the two layers at render time.

6. **Case sensitivity** — All tokens are case-sensitive. `{companyname}` will not be
   replaced. The only lowercase outlier is `{currentDate}`.

## Known Edge Cases

| Edge case | Behavior |
|---|---|
| `{currentDate}` — lowercase 'c' | Legacy from Tool 2. Preserved exactly. Do not rename. |
| `{ApprovedBy}` empty | Valid — renders as blank, document is "not yet approved" |
| `{Logo}` missing file | Non-fatal — renders as empty string, no image inserted |
| Token split across XML runs | Handled by `easy-template-x` automatically |
| Placeholder not in template | Silently ignored — value provided but token absent causes no error |
| Token in template but no value provided | Token appears literally in output — indicates a configuration error |

## Regression Template

`templates/test_render.docx` is the canonical regression template. It contains
placeholders from all categories above and must be used to validate:

- Body paragraph replacement
- Table cell replacement
- Header replacement
- Footer replacement
- Logo insertion
- Empty string replacement
- `{currentDate}` lowercase edge case

When the renderer (`shared/renderer/render.js`), `easy-template-x`, or logo handling
logic changes, run the regression test before committing:

```bash
# From project root
python3 templates/create_test_template.py
node shared/renderer/render.js /tmp/cert_expert_regression_test.json
# Open outputs/test_render_result.docx and verify visually
```
