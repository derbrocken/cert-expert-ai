# SK Veranstaltung — Gerüst (Praxis)

Checkliste für **SK-Gliederung** — keine Zahlen ohne Input.

---

## 1. Pflichtlogik SK (wann?)

SK sinnvoll wenn mindestens eines zutrifft (vgl. DGUV-Extrakt Crowd):

- Art/Größe der Veranstaltung
- unübersichtlicher oder temporärer Ort
- erwartetes Publikumsverhalten (nicht nur „friedlich“)
- Interaktion Publikum / erhöhte Bedrohung
- genehmigungsrechtliche Anforderungen im Input

Sonst: `[OFFENER PUNKT]` ob SK vom AG existiert.

---

## 2. Kapitelgerüst (SK_*)

| Block | Inhalt (Prinzip) |
|-------|------------------|
| Schutzziel | Was wird geschützt, Rahmen Event |
| Gefährdungsanalyse | Publikum, Lage, Organisation, Schnittstellen |
| Schutzmaßnahmen | Zonen, Einlass, Crowd, Eskalation, Ordnungsdienst |
| Verantwortliche | Veranstalter, EL SD, Behörden als Rolle |
| Offene Punkte | Vor Freigabe |

---

## 3. Prüffragen vor Generierung

1. Subtyp-MD geladen?
2. SK-Vorgabe des AG im Input?
3. Ordnungsdienst ja/nein belegt?
4. Fluchtwege/Kapazität nur mit Unterlage?

---

## 4. Abgrenzung GB/EK

- SK = **Gesamt**; GB = Arbeitsschutz SMA; EK = operative Umsetzung.

---

## 5. Stopps

- Keine Mindestkräfte, keine Genehmigung als erteilt.
