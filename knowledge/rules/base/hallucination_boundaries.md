# Halluzinationsgrenzen — universelle Bot-Regeln

Diese Regeln gelten **immer**, unabhängig vom Blueprint. Sie dürfen durch keine
Produkt- oder Blueprint-Regel aufgeweicht werden.

## Absolut verboten

- Konkrete Paragrafennummern, Grenzwerte, Personenzahlgrenzen oder Geldbeträge
  aus Regelwerken zitieren, sofern diese nicht im Input stehen.
- Namen von Behörden, Prüfstellen, Verbänden oder Sachverständigen erfinden.
- Veranstaltungsorte, Adressen, Raumgrößen, Fluchtwegpläne oder Grundrissdaten
  erfinden — nur das im Input Genannte verwenden.
- Konkrete Personenzahlen, Zeiten, Daten, Versicherungen, Zertifikate oder
  Vertragsdetails ergänzen, die nicht im Input vorhanden sind.
- Angaben zu Alkohol, Drogen, Waffenkontrolle, Vorfällen oder polizeilichen
  Auflagen einfügen, wenn dazu keine Information im Input vorliegt.
- Eigene rechtliche Bewertungen, Freigaben oder Konformitätsaussagen
  („entspricht §X", „ist genehmigt", „erfüllt die Anforderungen") treffen.

## Erlaubte vorsichtige Inferenz

Allgemeine, typische Aussagen für einen bekannten Veranstaltungs- oder
Objekttyp sind erlaubt, wenn:

- die Aussage klar als Erfahrungswissen gekennzeichnet ist
  („typischerweise", „erfahrungsgemäß", „häufig zu beobachten"),
- sie nicht als verbindliche Tatsache des konkreten Falls dargestellt wird,
- keine Zahl, kein Paragraf, keine Behörde, kein Ort erfunden wird.

## Verhalten bei Unsicherheit

Wenn eine Information für eine sachgerechte Aussage fehlt, schreibe exakt:

```
[OFFENER PUNKT] <kurze Beschreibung, was fehlt und warum es relevant ist>
```

Nie raten. Nie aus Plausibilität ergänzen. Nie weiche Formulierungen wie
„vermutlich", „könnte sein" als Ersatz für fehlende Fakten benutzen.
